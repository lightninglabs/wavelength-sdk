<Step title="Install">

Install the two packages you need: `walletdk-web` for the browser wallet runtime and `walletdk-react` for the React bindings.

```sh
npm install @lightninglabs/walletdk-web @lightninglabs/walletdk-react
```

Requires Node 20 or later to run locally. The runtime itself runs in Chrome 100+, Firefox 110+, and Safari 16+. No special build config is needed for Vite or Next.js.

</Step>

<Step title="Wire the provider">

Create a client with `createWebClient`, wrap your app with `WalletDKProvider`, and start the embedded daemon with `defaultConfig('signet')`. Every component below the provider gains access to the wallet through hooks.

```tsx title="App.tsx"
import { createWebClient, defaultConfig } from '@lightninglabs/walletdk-web';
import { WalletDKProvider, useWalletRuntime } from '@lightninglabs/walletdk-react';
import { useEffect } from 'react';

const client = createWebClient({ runtimeBaseUrl: '/walletdk/' });

function Boot() {
  const { phase, start } = useWalletRuntime();

  useEffect(() => {
    if (phase === 'runtimeReady') {
      start(defaultConfig('signet')).catch(() => {});
    }
  }, [phase, start]);

  return null;
}

export function App() {
  return (
    <WalletDKProvider client={client}>
      <Boot />
      <Balance />
      <SendButton />
    </WalletDKProvider>
  );
}
```

`defaultConfig(network, overrides?)` returns a `RuntimeConfig` preloaded with public gateway endpoints for that network, merged with any overrides you pass. See [Networks & config](/concepts/networks-and-config/) for the full list of `RuntimeConfig` fields.

Use `defaultConfig('mainnet')` only after you have key-backup UX in place and set `allowMainnet: true`.

</Step>

<Step title="Create a wallet">

Call `createWallet` from `useWalletBootstrap` to generate a new HD seed in the browser. The seed is encrypted locally and never transmitted.

```tsx title="Onboard.tsx"
import { useWalletBootstrap } from '@lightninglabs/walletdk-react';

function Onboard() {
  const { createWallet, operations } = useWalletBootstrap();

  const handleCreate = async () => {
    const { mnemonic } = await createWallet({ password: 'a-strong-password' });
    // Back up `mnemonic` securely before the user sends funds.
    console.log('Wallet created');
  };

  return (
    <button onClick={handleCreate} disabled={operations.createWallet.busy}>
      Create wallet
    </button>
  );
}
```

Once `createWallet` resolves, `phase` from `useWalletRuntime` automatically becomes `'ready'`, so any component watching `phase` updates without extra wiring.

</Step>

<Step title="Try it">

Call `send` with a Lightning invoice or on-chain address. WalletDK routes Lightning payments through the swap server and settles on-chain sends through a cooperative leave with the Ark operator.

```tsx title="SendButton.tsx"
import { useSend } from '@lightninglabs/walletdk-react';

function SendButton() {
  const { send, busy } = useSend();

  const handleSend = async () => {
    const result = await send({
      invoice: 'lnbc1…', // or onchainAddress: 'bc1q…'
    });
    console.log('Sent!', result.paymentHash ?? result);
  };

  return (
    <button onClick={handleSend} disabled={busy}>
      Send
    </button>
  );
}
```
