<Callout type="note" title="Pre-release">
The React Native SDK is pre-release and not yet published to npm. The API
surface is stable; iOS passkey support is still experimental.
</Callout>

<Step title="Install">

Install the two packages you need: `walletdk-react-native` for the native wallet runtime and `walletdk-react` for the React bindings.

```sh
npm install @lightninglabs/walletdk-react-native @lightninglabs/walletdk-react
```

Requires React Native 0.76+ with the New Architecture enabled, iOS 15.1+, and Android minSdk 24. Expo apps need a development build, not Expo Go: the native wallet runtime is a compiled module that Expo Go cannot load. See [Requirements](/react-native/get-started/requirements/) for the full platform matrix.

Until hosted binary distribution ships, the native wallet runtime must be staged before the first build; see [Installation](/react-native/get-started/installation/) for the staging step. The native side is built by `npx expo run:android` / `npx expo run:ios` (which installs iOS pods automatically), so there is no separate pod step to run by hand.

</Step>

<Step title="Wire the provider">

Pass `createNativeClient` to `WalletDKProvider` as its `createClient` factory, and start the embedded daemon with `defaultConfig('signet')`. The provider calls the factory once and memoizes the client, so the client is created when the provider mounts rather than at module import time. Every component below the provider gains access to the wallet through hooks.

```tsx title="App.tsx"
import { useEffect } from 'react';
import { createNativeClient, defaultConfig } from '@lightninglabs/walletdk-react-native';
import { WalletDKProvider, useWalletRuntime } from '@lightninglabs/walletdk-react';

function Boot() {
  const { phase, start } = useWalletRuntime();

  useEffect(() => {
    if (phase === 'runtimeReady') {
      start(defaultConfig('signet'));
    }
  }, [phase, start]);

  return null;
}

export default function App() {
  return (
    <WalletDKProvider createClient={createNativeClient}>
      <Boot />
      {/* your app */}
    </WalletDKProvider>
  );
}
```

`defaultConfig(network, overrides?)` returns a `RuntimeConfig` preloaded with public gateway endpoints for that network, merged with any overrides you pass. See [Networks & config](/concepts/networks-and-config/) for the full list of `RuntimeConfig` fields.

Use `defaultConfig('mainnet')` only after you have key-backup UX in place and set `allowMainnet: true`.

</Step>

<Step title="Create a wallet">

Call `createWallet` from `useWalletBootstrap` to generate a new HD seed on the device. The seed is encrypted locally and never transmitted.

```tsx title="Onboard.tsx"
import { Button } from 'react-native';
import { useWalletBootstrap } from '@lightninglabs/walletdk-react';

function Onboard() {
  const { createWallet, operations } = useWalletBootstrap();

  const handleCreate = async () => {
    const { mnemonic } = await createWallet({ password: 'a-strong-password' });
    // Back up `mnemonic` securely before the user sends funds.
    console.log('Wallet created');
  };

  return (
    <Button
      title="Create wallet"
      onPress={handleCreate}
      disabled={operations.createWallet.busy}
    />
  );
}
```

Once `createWallet` resolves, `phase` from `useWalletRuntime` automatically becomes `'ready'`, so any component watching `phase` updates without extra wiring.

</Step>

<Step title="Try it">

Call `send` with a Lightning invoice or on-chain address. WalletDK routes Lightning payments through the swap server and settles on-chain sends through a cooperative leave with the Ark operator.

```tsx title="SendButton.tsx"
import { Button } from 'react-native';
import { useSend } from '@lightninglabs/walletdk-react';

function SendButton() {
  const { send, busy } = useSend();

  const handleSend = async () => {
    const result = await send({
      invoice: 'lnbc1…', // or onchainAddress: 'bc1q…'
    });
    console.log('Sent!', result.paymentHash ?? result);
  };

  return <Button title="Send" onPress={handleSend} disabled={busy} />;
}
```

You are all set. Run the app on a device or simulator to see it end to end; see [Installation](/react-native/get-started/installation/) for staging the native runtime binaries and getting a build running.

</Step>
