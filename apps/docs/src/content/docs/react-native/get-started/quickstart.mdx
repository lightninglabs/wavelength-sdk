<Callout type="note" title="Pre-release">
The React Native SDK is pre-release and not yet published to npm. The API
surface is stable; iOS passkey support is still experimental.
</Callout>

<Step title="Install">

Install the two packages you need: `walletdk-react-native` for the native wallet runtime and `walletdk-react` for the React bindings.

```sh
npm install @lightninglabs/walletdk-react-native @lightninglabs/walletdk-react
```

Requires React Native 0.76+ with the New Architecture enabled, iOS 15.1+, and Android minSdk 24. Expo apps need a development build, not Expo Go: the native wallet runtime is a compiled module that Expo Go cannot load. See [Requirements](/react-native/get-started/requirements/) for the full platform matrix.

Until hosted binary distribution ships, the native wallet runtime must be staged before the first build; see [Installation](/react-native/get-started/installation/) for the staging step. The native side is built by `npx expo run:android` / `npx expo run:ios` (which installs iOS pods automatically), so there is no separate pod step to run by hand.

</Step>

<Step title="Wire the provider">

Create a `WalletEngine` with `createNativeWalletEngine`, and wrap your app with `WalletDKProvider`. Pass `config` and `autoStart: true` to boot the embedded daemon as soon as the runtime is ready, no boot effect needed. Every component below the provider gains access to the wallet through hooks.

```tsx title="App.tsx"
import { Text } from 'react-native';
import { createNativeWalletEngine } from '@lightninglabs/walletdk-react-native';
import { WalletDKProvider, useWallet } from '@lightninglabs/walletdk-react';
import { defaultConfig } from '@lightninglabs/walletdk-core';

const engine = createNativeWalletEngine({
  config: defaultConfig('signet'),
  autoStart: true,
});

export default function App() {
  return (
    <WalletDKProvider engine={engine}>
      <Status />
      {/* your app */}
    </WalletDKProvider>
  );
}

function Status() {
  const { phase } = useWallet();

  return <Text>Wallet phase: {phase}</Text>;
}
```

`defaultConfig(network, overrides?)` returns a `RuntimeConfig` preloaded with public gateway endpoints for that network, merged with any overrides you pass. See [Networks & config](/concepts/networks-and-config/) for the full list of `RuntimeConfig` fields.

Use `defaultConfig('mainnet')` only after you have key-backup UX in place and set `allowMainnet: true`.

</Step>

<Step title="Create a wallet">

Call `create` from `useWalletCreate` to generate a new HD seed on the device. The seed is encrypted locally and never transmitted.

```tsx title="Onboard.tsx"
import { Button } from 'react-native';
import { useWalletCreate } from '@lightninglabs/walletdk-react';

function Onboard() {
  const { create, createPending } = useWalletCreate();

  const handleCreate = async () => {
    const { mnemonic } = await create({ password: 'a-strong-password' });
    // Back up `mnemonic` securely before the user sends funds.
    console.log('Wallet created');
  };

  return (
    <Button
      title="Create wallet"
      onPress={handleCreate}
      disabled={createPending}
    />
  );
}
```

Once `create` resolves, `phase` from `useWallet` automatically advances (through `'syncing'` when the daemon needs to catch up) to `'ready'`, so any component watching `phase` updates without extra wiring.

</Step>

<Step title="Try it">

Call `send` from `useWalletSend` with a Lightning invoice or on-chain address. WalletDK routes Lightning payments through the swap server and settles on-chain sends through a cooperative leave with the Ark operator.

```tsx title="SendButton.tsx"
import { Button } from 'react-native';
import { useWalletSend } from '@lightninglabs/walletdk-react';

function SendButton() {
  const { send, sendPending } = useWalletSend();

  const handleSend = async () => {
    const result = await send({
      invoice: 'lnbc1…', // or onchainAddress: 'bc1q…'
    });
    console.log('Sent!', result.paymentHash ?? result);
  };

  return <Button title="Send" onPress={handleSend} disabled={sendPending} />;
}
```

You are all set. Run the app on a device or simulator to see it end to end; see [Run the demo app](/react-native/get-started/run-the-demo-app/) for staging the native runtime binaries and getting a build running.

</Step>
