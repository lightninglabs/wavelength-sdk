`swap` operates on Lightning swap sessions via the daemon-owned swap client
service. The commands are thin RPC renderers: swap execution continues inside
`darepod` after the command exits, so these verbs create, inspect, and resume
sessions rather than driving them to completion in-process.

<Callout type="caution" title="Requires a swapruntime build">
Swap execution is daemon-owned, so the CLI only exposes these commands when
`darepocli` is built with the `swapruntime` build tag. In a stub build the
`swap` command is a discoverable placeholder that returns an error instructing
you to rebuild with `-tags=swapruntime`.
</Callout>

## Flags

`swap` itself takes no bespoke flags; it dispatches to one of the subcommands
below. Every subcommand accepts `--help` and the global connection flags
(`--rpcserver`, `--tlscertpath`, `--no-tls`).

## Example

`swap` alone prints usage and exits; pick a subcommand, for example:

```bash title="Terminal"
darepocli swap list --pending
```

## Subcommands

### `swap list`

Lists persisted Lightning swap sessions.

| Flag | Default | Description |
|---|---|---|
| `--pending` | `false` | Show only non-terminal resumable swaps. |

Calls the daemon `ListSwaps` RPC.

```bash title="Terminal"
darepocli swap list --pending
```

### `swap show`

Shows one persisted swap by payment hash. The payment hash can come from the
positional argument or from `--json`.

| Flag | Default | Description |
|---|---|---|
| `<payment_hash>` | (none) | The swap payment hash. Required unless supplied via --json. |

Calls the daemon `GetSwap` RPC.

```bash title="Terminal"
darepocli swap show <payment_hash>
```

### `swap receive`

Creates a receive swap intent: receive BTC over Lightning into Ark. The command
returns once the daemon has durably persisted the invoice session and taken
ownership of the background receive worker.

| Flag | Default | Description |
|---|---|---|
| `--amount` | (none) | Amount in satoshis to receive. Required and must be positive; or pass via --json. |

Calls the daemon `StartReceive` RPC.

```bash title="Terminal"
darepocli swap receive --amount 50000
```

### `swap pay`

Creates a pay swap intent: pay a Lightning invoice from Ark funds. Once the RPC
returns, the daemon owns funding, preimage observation, refund, and
terminalization for that payment hash.

| Flag | Default | Description |
|---|---|---|
| `--invoice` | (none) | BOLT-11 Lightning invoice to pay. Required; or pass via --json. |
| `--maxfee` | `0` | Maximum swap fee in satoshis. 0 lets the daemon cap the fee at roughly 1% of the amount. |

Calls the daemon `StartPay` RPC.

```bash title="Terminal"
darepocli swap pay --invoice lnbcrt250u1p3xyz...
```

### `swap resume`

Manually wakes a persisted swap. The daemon deduplicates by payment hash, so
resume cannot create a second driver for an already active swap. Both the
payment hash and the direction are required (from the positional argument and
`--direction`, or via `--json`).

| Flag | Default | Description |
|---|---|---|
| `<payment_hash>` | (none) | The swap payment hash. Required unless supplied via --json. |
| `--direction` | (none) | Swap direction to resume: pay or receive. Required; or pass via --json. |

Calls the daemon `ResumeSwap` RPC.

```bash title="Terminal"
darepocli swap resume <payment_hash> --direction pay
```

### `swap watch`

Streams swap updates. It may emit existing swap rows first, then live
worker-exit updates as the daemon reports them.

| Flag | Default | Description |
|---|---|---|
| `--pending` | `false` | Stream only pending swap updates. |
| `--include-existing` | `true` | Emit existing swaps before live updates. |

Calls the daemon `SubscribeSwaps` RPC.

```bash title="Terminal"
darepocli swap watch
```
