## The three gateways

Starting the embedded daemon requires three REST endpoints your app never calls directly, but they must all serve the same Bitcoin network. Misaligned URLs are a common cause of stuck deposits or swaps that never fund.

The embedded daemon never talks to Bitcoin or Ark nodes directly from your app code. `RuntimeConfig` is the shared `core` contract, so the same fields apply on either transport: after `start()`, the daemon reaches these **three REST gateways**. (The React Native transport reaches the daemon over its native gRPC; the web transport uses REST because browsers cannot dial native gRPC. Either way these three gateway URLs are what the daemon connects out to.)

1. **`arkServerUrl`** (Ark operator + mailbox edge): rounds, boarding, VTXO relay, cooperative leave, and mailbox traffic share this gateway. There is no separate mailbox URL in the launch config.
2. **`swapServerUrl`** (swap server): Lightning↔Ark atomic swaps (Lightning receive and send).
3. **`esploraUrl`** (Esplora): chain queries and UTXO lookups for boarding detection. Must implement the Esplora `/address/:addr/utxo` API (electrs / mempool.space compatible).

All three must point at infrastructure that serves the same **`network`**. A mismatched Esplora endpoint or swap server is a common source of "stuck deposit" or "swap never funds" bugs.

## defaultConfig presets

For public test networks, use your transport package's `defaultConfig(network)` instead of hand-typing URLs. It returns a ready **RuntimeConfig** with canonical endpoints merged with any overrides you pass:

<PlatformTabs>
<div data-label="Web">

```ts
import { createWebClient, defaultConfig } from '@lightninglabs/wavelength-web';

const client = createWebClient();
const config = defaultConfig('signet', { dataDir: 'my-app-wallet' });
await client.start(config);
```

</div>
<div data-label="React Native">

```ts
import { createNativeClient, defaultConfig } from '@lightninglabs/wavelength-react-native';

const client = createNativeClient();
const config = defaultConfig('signet', { dataDir: 'my-app-wallet' });
await client.start(config);
```

</div>
</PlatformTabs>

Presets ship for:

| Network | Typical use |
|---------|-------------|
| `'signet'` | Default for docs live embeds and the demo app; public Lightning Labs test infrastructure. |
| `'testnet'` | Bitcoin testnet3 with hosted Ark/swap gateways. |
| `'testnet4'` | Bitcoin testnet4 with hosted Ark/swap gateways. |

Hosted test-network gateways use TLS; on the web transport they follow a `*-rest` hostname pattern, while the React Native transport dials the matching host:port gRPC addresses. `defaultConfig` accepts only these preset networks. `'mainnet'` and `'regtest'` are excluded: mainnet has no public deployment yet, and regtest's local ports vary per development environment. Build a `RuntimeConfig` for either by hand: mainnet with your own gateway URLs and `allowMainnet` (see below), regtest with local URLs and the insecure-transport flags.

Pass overrides as the second argument to change data directory, point at your own operator, or toggle advanced flags without losing the rest of the preset.

## allowMainnet

Mainnet is deliberately gated. Set **`allowMainnet: true`** together with **`network: 'mainnet'`** and your own **`arkServerUrl`**, **`swapServerUrl`**, and **`esploraUrl`**. Without **`allowMainnet`**, the daemon refuses to start on mainnet even if URLs are present.

<Callout type="caution">
`allowMainnet` is an explicit safety rail. Do not enable it in example code or demos that users might copy against production networks by accident.
</Callout>

Test networks ignore **`allowMainnet`**; it exists only for mainnet runs.

## advanced options

Most apps only need `defaultConfig(network)` plus an optional data directory. The remaining **RuntimeConfig** fields are for self-hosting, local dev, or specialized deployments:

| Field | Purpose |
|-------|---------|
| **`dataDir`** | Storage root for daemon and wallet state (an OPFS path in the browser; a filesystem path in the app data directory on React Native). |
| **`disableSwaps`** | Turn off the Lightning swap subsystem entirely; Ark boarding and leaves still work. |
| **`serverInsecure`** | Skip TLS verification for **`arkServerUrl`** (local dev only). |
| **`swapServerInsecure`** | Skip TLS verification for **`swapServerUrl`** (local dev only). |
| **`swapDatabaseFileName`** | Daemon-owned SQLite file for swap session state. |
| **`debugLevel`** | Daemon log verbosity (distinct from the web client's RPC **`debug`** flag). |

On regtest, set both insecure flags yourself because local gateways speak plain HTTP. On the hosted networks, leave them unset so connections stay TLS-verified.

If you self-host Ark or swap infrastructure, replace the preset URLs while keeping network aligned with your Esplora instance. Setting `disableSwaps: true` is appropriate for boarding-only experiments where Lightning is out of scope. Full field documentation is in the [wavelength-core reference](/reference/wavelength-core/).
